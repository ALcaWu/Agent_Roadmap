### 📚 学习内容
- **核心概念**：
  - **async/await**: 定义协程和挂起点（类比 Kotlin 的 suspend/await）
  - **事件循环**: Python 异步的核心调度机制
  - **并发 vs 并行**: asyncio 是单线程并发，适合 IO 密集型任务
  - **asyncio.gather**: 并发启动多个任务并等待全部完成（类比 Kotlin 的 awaitAll）

- **实践代码**：
  - `src/day4_async/async_agent_tools.py`: 异步工具执行器，模拟 Agent 并发调用搜索、数据库和计算工具。

- **技术对比 (Kotlin vs Python)**:
  | Kotlin | Python | 说明 |
  | :--- | :--- | :--- |
  | `suspend fun` | `async def` | 定义可挂起函数 |
  | `delay()` | `asyncio.sleep()` | 非阻塞延迟 |
  | `awaitAll()` | `asyncio.gather()` | 并发执行 |
  | `runBlocking` | `asyncio.run()` | 启动入口 |

### 💡 学习收获
1. **思维迁移**：
   - 发现 Python 的异步编程模型与 Kotlin Coroutines 惊人地相似。
   - 理解了 `await` 就是交出控制权的时刻，让 CPU 去处理其他任务。

2. **Agent 性能优化**：
   - 明白了为什么 Agent 必须异步：同时调用 LLM、向量库和外部 API 能大幅降低总延迟。
   - 如果不使用 `gather`，Agent 的响应时间将是所有工具耗时的总和；使用后，接近最慢工具的耗时。

3. **代码规范**：
   - 养成了添加详细注释的习惯（每行注释 + Kotlin 类比）。
   - 学会了使用 `aiohttp` 替代 `requests` 进行异步网络请求。

### 🐛 遇到的问题 & 解决
- **问题**: 刚开始对 `asyncio.run()` 和 `await` 的区别有点模糊。
- **解决**: 通过代码实践理解了 `run()` 是启动整个循环的入口，而 `await` 是循环内部的挂起点。
