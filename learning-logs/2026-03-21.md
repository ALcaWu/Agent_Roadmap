### 📚 学习内容
- **核心概念**：
  - **Transformer**: LLM 的基础架构，核心是 Self-Attention（自注意力机制），让模型能关注上下文。
  - **Tokenization**: 文本的最小单位。OpenAI 按 Token 收费。
    - 英文：1词 ≈ 0.75-1 Token
    - 中文：1字 ≈ 1-2 Token (成本更高)
  - **Embeddings**: 将文本转换为高维向量（数字列表），语义相近的文本在向量空间中距离更近。

### 💡 学习收获
1. **LLM 工作原理**：
   - LLM 本质上是**概率模型**，它不是真的“懂”逻辑，而是基于海量文本统计出下一个 Token 出现的概率。
   - Transformer 架构通过“注意力”机制解决了长距离依赖问题。

2. **成本意识**：
   - 以后写 Prompt 时要精简，因为 Token 就是钱。
   - 代码压缩可以显著节省 Token（因为代码的 Token 密度通常比自然语言低）。

3. **RAG 原理初探**：
   - 搜索不再是简单的关键词匹配（如 SQL `LIKE`），而是向量空间中的数学计算（余弦相似度）。
   - 这就是为什么 Agent 能回答训练数据之外的问题——因为它检索了外部知识库的向量。
